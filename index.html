1. Core Architecture & Algorithms

Multi-stage pipeline converting raw video into actionable cheating alerts.

  • Feature Extraction: MediaPipe FaceMesh (Refined Landmarks enabled) extracting 468 3D facial landmarks
  • Model Type: Custom Deep Feedforward Neural Network (Multi-Layer Perceptron) built in PyTorch
  • Input Layer: 936 neurons (468 landmarks x 2 coordinates X, Y)
  • Hidden Layer 1: 1024 neurons
  • Hidden Layer 2: 512 neurons
  • Hidden Layer 3: 256 neurons
  • Hidden Layer 4: 128 neurons
  • Output Layer: 3 neurons (Pitch, Yaw, Roll)
  • Activation: ReLU after every hidden layer
  • Stabilization: BatchNorm1d for training stability
  • Regularization: Dropout (30% down to 10%)
  • Optimization: AdamW optimizer
  • Loss Function: SmoothL1Loss (Huber Loss) to handle spatial outliers
  • Preprocessing: StandardScaler (Scikit-Learn) for distance-invariance
2. Dataset & Training Details
  • Dataset Source: 300W-LP dataset
  • Sample Count: 4,286 high-quality samples
  • Hardware: Dual Tesla T4 GPU cluster (T4*2) using nn.DataParallel
  • Batch Size: 256 images
  • Scheduler: ReduceLROnPlateau
  • Early Stopping: 100 epochs max, halt if validation plateaus for 15 rounds
  • Inference Speed: 30+ FPS on standard local CPU
3. The Problem Solved

Privacy-Preserving Exam Integrity detection system.

  • Head Pose Deviations: Detects looking away from the screen (Pitch/Yaw deviations)
  • Eye Gaze Shifts: Side-eyeing notes or devices using mathematical Iris Tracking ratio
  • Privacy: No data leaves the device; entire 200MB system runs locally

5. Engineering Logic (Post-Model)
  • Temporal Smoothing: 5-frame buffer prevents flickering alarms from micro-movements
  • Personal Calibration: 30-frame initialization sets student's natural posture as (0,0) baseline
  • Pure-Math Eye Tracker: Geometric algorithm using MediaPipe Iris tracker (ratio between pupil and eye corners)
  • Thresholds: Flags "side-eye" if ratio below 0.38 or above 0.62
  • Offline Security: No internet required; runs in Python virtual environment
  • System Footprint: Less than 200MB storage

2. Dataset & Temporal Specifications
  • Dataset Source: MSU Online Exam Proctoring (OEP) dataset from Michigan State University Computer Vision Lab
  • Volume: 560 video recordings
  • Composition: 24 participants (15 actors simulating cheating, 9 students in real exam scenarios)
  • Annotation: Start time, end time, and specific label for cheating behavior type
  • Temporal Logic: Videos decomposed into RGB frames; specific number of frames (n) extracted for multidimensional matrix
  • Sequence Creation: Buffer of 30 frames for temporal window
  • Data Imbalance: 284 talking instances vs 15 phone call instances
  • Handling: Deep transfer learning manages imbalance without data resampling
3. Operational Efficiency (Training vs Testing)
  • Training Time: ResNet50V2-GRU faster to train than ResNet50V2-LSTM variant
  • Testing Time (Latency): Superior classification effectiveness and faster testing times than all other tested models
  • CPU Performance: Approximately 0.3752 seconds for testing phase
4. Algorithmic Methodology
A. Feature Extraction: ResNet50V2
  • 50-layer deep Convolutional Neural Network with "skip connections"
  • Solves degradation problem in very deep networks
  • Transfer Learning with ImageNet weights
  • Most robust among VGG16 and EfficientNet
  • Recognizes complex patterns: hands, phones, paper textures
B. Temporal Analysis: GRU (Gated Recurrent Unit)
  • Simplified version of LSTM using update and reset gates
  • Manages sequential information flow
  • Analyzes sequence of frames for action recognition
  • Significantly faster in training and testing than LSTM version

Model 3: Real-Time Multi-Brain Proctoring Engine(advancedd laptop cam)

Integrated 5-AI Brain System | Flask + OpenCV + Threading Architecture
Download Notebook
1. The Core Infrastructure (The Factory Floor)

System architecture for moving data around before AI processing begins.

  • Flask (The Web Server): Handles web page (dashboard.html), creates buttons, opens constant pipe to stream video to browser
  • OpenCV (The Eyes): Ultimate computer vision library; talks to webcam, grabs picture 30 times per second, hands to AI models
  • Threading: Crucial concept; puts Audio and Screen Capture on separate background workers so main video feed never freezes
Architecture: Video processing is fast, but audio and screen recording can cause lag. Separate threads prevent system bottlenecks.
2. The 5 AI "Brains" (The Algorithms)

Every webcam frame passes down an assembly line to these five models:

Brain A: Face & Multi-Face Detection

MTCNN

Multi-task Cascaded Convolutional Networks using three neural networks in a row. First scans whole image for face-like regions, second filters mistakes, third pinpoints exact bounding box.

Logic: Count boxes. If count == 0, trigger "Face Disappeared". If count > 1, trigger "Multiple Faces".

Brain B: Eye Tracking

Google Face Mesh

Highly optimized neural network mapping 468 specific 3D coordinates. Knows exact coordinates for nose tip, eye corners, and facial features.

Logic: Compare eye coordinates to nose tip. If mathematical distance shifts too far left/right, trigger "Gaze Away".

Brain C: Mouth Tracking

Google Face Mesh

Same 468-point MediaPipe network focused on lip region. Tracks vertical distance between upper lip (point 13) and lower lip (point 14).

Logic: Calculate vertical gap between lips. If gap exceeds threshold, trigger "Mouth Moving".

Brain D: Forbidden Object Detection

YOLOv8 Nano

You Only Look Once - looks at entire image exactly once through massive CNN. Divides image into grid, predicts bounding boxes and probabilities.

Logic: Ignore cars and dogs. If Class 67 (Cell Phone) or Class 73 (Book) detected with confidence > 65%, draw red box and trigger alert.

Brain E: Audio Monitoring

ZCR / Energy + Whisper

Two-stage: First fast math (Zero-Crossing Rate and Audio Energy) detects volume spikes. Second stage passes noise to OpenAI Whisper, a Transformer-based network converting audio to text.

Logic: If volume spikes above threshold, trigger "Voice Detected".

3. The Support Crew (The Utilities)

Files handling paperwork once brains detect cheating:

  • logging.py: Acts as database. Writes exact timestamp and violation type to text file (alerts.log). Uses cooldown timer so 10-second phone hold logs once, not 300 times.
  • alert_system.py: Uses gTTS (Google Text-to-Speech) to generate mp3 saying "Please look at the screen", plays instantly via Pygame through speakers.
  • screen_capture.py: Uses MSS for rapid screenshots, stitches into .mp4 video using OpenCV, proving student wasn't Googling answers.
  • report_generator.py: Reads alerts.log on "Stop Exam" click. Uses Matplotlib for mathematical graphs (heatmaps of cheating frequency), compiles into PDF via wkhtmltopdf.
4. The 30-Millisecond Lifecycle

What happens 30 times every second while app runs:

Step 1: Capture

OpenCV grabs a picture from webcam

Step 2: Face Check

MTCNN checks if you are present and counts faces

Step 3: Gaze Check

MediaPipe checks where eyes are looking

Step 4: Mouth Check

MediaPipe checks if mouth is open/talking

Step 5: Object Check

YOLO checks for phones and books

Step 6: Alert & Stream

If violation found, Logger writes it down and Pygame yells. OpenCV encodes to .jpg and streams to HTML dashboard

Step 7: Repeat

Cycle restarts immediately

Speed: Entire pipeline completes in approximately 30 milliseconds, enabling real-time 30 FPS monitoring with instant violation detection and alerting.
